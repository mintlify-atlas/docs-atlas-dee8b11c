## Authentication Components

### DiscordButton

OAuth button for Discord authentication.

<ParamField path="className" type="string" default="">
  Additional CSS classes
</ParamField>

<ParamField path="onlyWrapper" type="boolean" default="false">
  If true, renders only the anchor wrapper without button styles
</ParamField>

<ParamField path="children" type="React.ReactNode">
  Custom button content (defaults to Discord icon + "Entrar na Comunidade")
</ParamField>

```tsx
import DiscordButton from "~/components/features/auth/discord-button";

// Default usage
<DiscordButton />

// Custom content
<DiscordButton className="w-full">
  <BsDiscord className="w-4 h-4 mr-2" />
  <span>Join Discord</span>
</DiscordButton>

// As wrapper only
<DiscordButton onlyWrapper className="custom-link">
  <CustomButton>Connect Discord</CustomButton>
</DiscordButton>
```

**OAuth Flow:**
The component automatically constructs the Discord OAuth URL with:
- Client ID: `1185302801178439780`
- Redirect URI: `{BASE_URL}/auth/discord/callback`
- Scopes: `identify guilds.join email`

## Comment System

### CommentSection

Complete comment section with replies, editing, and deletion.

<ParamField path="comments" type="Comment[]" required>
  Array of comment objects
</ParamField>

<ParamField path="commentableId" type="string | number" required>
  ID of the resource being commented on
</ParamField>

<ParamField path="commentableType" type="'ChallengeUser' | 'Lesson'" required>
  Type of resource being commented on
</ParamField>

<ParamField path="redirectTo" type="string" required>
  URL to redirect to after login (if not authenticated)
</ParamField>

```tsx
import CommentSection from "~/components/features/comments/comment-section";
import type { Comment } from "~/lib/models/comments.server";

export default function LessonPage({ comments, lesson }) {
  return (
    <div>
      {/* Lesson content */}
      
      <CommentSection
        comments={comments}
        commentableId={lesson.id}
        commentableType="Lesson"
        redirectTo={`/workshops/${lesson.slug}`}
      />
    </div>
  );
}
```

### Optimistic Updates

The component uses optimistic UI updates for instant feedback:
```tsx
const optimisticToggleReaction = (reaction: AllowedReaction) => {
  if (localUserReacted.includes(reaction)) {
    // Remove reaction
    setTotalReactions((prev) => prev! - 1);
    setLocalReactions((prev) => ({
      ...prev,
      [reaction]: (prev[reaction] || 0) - 1,
    }));
  } else {
    // Add reaction
    setTotalReactions((prev) => prev! + 1);
    setLocalReactions((prev) => ({
      ...prev,
      [reaction]: (prev[reaction] || 0) + 1,
    }));
  }
};
```

## Form Components

### LoadingButton

Button with loading and success states.

<ParamField path="status" type="'idle' | 'loading' | 'submitting'" required>
  Current button state
</ParamField>

<ParamField path="isSuccessfulSubmission" type="boolean" default="false">
  Show success checkmark
</ParamField>

<ParamField path="size" type="ButtonSize" default="default">
  Button size variant
</ParamField>

```tsx
import LoadingButton from "~/components/features/form/loading-button";
import { useNavigation } from "react-router";

export default function MyForm() {
  const navigation = useNavigation();
  const isSubmitting = navigation.state === "submitting";
  
  return (
    <Form method="post">
      <Input name="email" type="email" />
      
      <LoadingButton 
        type="submit"
        status={isSubmitting ? "submitting" : "idle"}
        size="lg"
      >
        Submit Form
      </LoadingButton>
    </Form>
  );
}
```

## Best Practices

### Authentication Required Features

Always check for user authentication before rendering interactive features:

```tsx
import { useOutletContext } from "react-router";
import type { User } from "~/lib/models/user.server";

export default function MyPage() {
  const { user } = useOutletContext<{ user: User | null }>();
  
  return (
    <div>
      {user ? (
        <CommentSection {...props} />
      ) : (
        <Button onClick={() => navigate("/login")}>
          Login to Comment
        </Button>
      )}
    </div>
  );
}
```

### Optimistic UI Updates

Use optimistic updates for better UX with immediate feedback:

```tsx
const handleReaction = async (reaction) => {
  // Update UI immediately
  setLocalReaction(reaction);
  
  // Submit to server
  fetcher.submit({ reaction }, { method: "post" });
};
```

### Error Handling

Always handle errors gracefully with user-friendly messages:

```tsx
useEffect(() => {
  if (fetcher.data?.error) {
    toast.error(fetcher.data.error);
  }
}, [fetcher.data]);
```
