## Overview

Codeante.io provides professional certificates for completed workshops and challenges. Certificates validate your learning achievements and can be shared on LinkedIn, portfolios, and resumes.

## Certificate Structure

```typescript
type Certificate = {
  id: string;
  user: User;
  certifiable_type: "ChallengeUser" | "WorkshopUser";
  certifiable_id: string;
  certifiable: Certifiable;
  status: "pending" | "published";
  metadata: CertificateMetadata;
  created_at: string;
  error?: any;
}
```

### Certificate Types

<Tabs>
  <Tab title="Challenge Certificates">

## Certificate Metadata

Certificates contain rich metadata for validation:

```typescript
type CertificateMetadata = ChallengeUserMetadata | WorkshopUserMetadata;
```

### Common Fields

- **certifiable_source_name**: Title of the challenge or workshop
- **certifiable_slug**: Unique identifier for the content
- **end_date**: Completion date
- **start_date**: When you began (challenges only)

### Challenge-Specific Fields

```typescript
tags: string[];  // Technologies and frameworks used
```

## Requesting Certificates

Certificates must be requested after completing content:

```typescript
await requestCertificate(
  request,
  certifiable_type,
  certifiable_id
);
```

<Steps>
  <Step title="Complete Content">
    Finish all requirements for the challenge or workshop
    
    **For Challenges:**
    - Join the challenge
    - Fork the repository
    - Submit your solution
    - Mark as completed
    
    **For Workshops:**
    - Watch all lessons
    - Complete associated exercises
    - Status marked as "completed"
  </Step>
  
  <Step title="Request Certificate">
    Click the certificate request button on the completion page
    
    ```typescript
    certifiable_type: "ChallengeUser" | "WorkshopUser"
    certifiable_id: string;  // Your user association ID
    ```
  </Step>
  
  <Step title="Certificate Generation">
    Certificate is generated and status updates from "pending" to "published"
    
    ```typescript
    status: "pending" | "published"
    ```
  </Step>
</Steps>

## Certificate Status

<Tabs>
  <Tab title="Pending">

    ```typescript

    ```
    
    <Info>

  <Tab title="Published">


## Accessing Certificates

### By Challenge Slug

```typescript
await getCertificateBySlug(request, slug);
// Returns: Certificate for specific challenge
```

### By Certificate ID

```typescript
await getCertificateById(id);
// Returns: Full certificate object
```

<Info>
Certificates can be accessed via direct URL: `/certificates/{certificate-id}`
</Info>

## Certificate Display

Certificates include:

- **User Name**: Your full name
- **Course/Challenge Title**: What you completed
- **Completion Date**: When you finished
- **Technologies**: Tags showing tech stack (challenges)
- **Duration**: Time invested (workshops)
- **Certificate ID**: Unique verification number
- **Codante Logo**: Official branding
