## Comments

The commenting system allows threaded discussions on submissions and content.

### Comment Structure

```typescript
type Comment = {
  id: string;
  user: User;
  comment: string;
  replying_to?: string;
  commentable_id: string;
  commentable_type: string;
  created_at_human: string;
}
```

### Creating Comments

```typescript
await createComment(
  request,
  commentableId,
  commentableType,
  comment,
  replyingTo
);
```

<Steps>
  <Step title="Select Content">
    Comments can be added to challenges, workshops, and submissions
    
    ```typescript
    commentable_type: "Challenge" | "Workshop" | "ChallengeUser"
    ```
  </Step>
  
  <Step title="Write Comment">
    Share insights, ask questions, or provide feedback
  </Step>
  
  <Step title="Optional Reply">
    Reply to existing comments for threaded discussions
    
    ```typescript
    replying_to?: string;  // Parent comment ID
    ```
  </Step>
</Steps>

### Managing Comments

<Tabs>
  <Tab title="Edit">
    Update your existing comments:
    
    ```typescript
    await updateComment(request, commentId, newComment);
    // Returns: { success } | { error: "Não foi possível editar o comentário." }
    ```
  </Tab>
  
  <Tab title="Delete">
    Remove your comments:
    
    ```typescript
    await deleteComment(request, commentId);
    // Returns: { success } | { error: "Não foi possível deletar o comentário." }
    ```
  </Tab>
</Tabs>

<Note>
You can only edit or delete your own comments. Admin users may have additional moderation capabilities.
</Note>

### Comment Display

Comments include:
- **User Avatar**: Profile picture with badge (PRO/Admin)
- **Username**: GitHub handle with link
- **Timestamp**: Human-readable time (e.g., "2 hours ago")
- **Content**: Formatted comment text
- **Actions**: Edit/delete for own comments, reply for all

## Reactions

React to submissions and content with emoji reactions.

### Reaction Types

```typescript
type AllowedReaction = "like" | "exploding-head" | "rocket" | "fire";
```

  <Card title="Like" icon="thumbs-up">

  <Card title="Exploding Head" icon="face-exploding">

  </Card>
  <Card title="Rocket" icon="rocket">

  <Card title="Fire" icon="fire">

  </Card>

### Reaction Data

```typescript
type Reactions = {
  reaction_counts: ReactionCount[];
  user_reactions: AllowedReaction[];
}

type ReactionCount = {
  reaction: AllowedReaction;
  count: number;
}
```

### Adding Reactions

```typescript
await react(
  request,
  reactableId,
  reactableType,
  reaction
);
```

<Info>
Reactions can be added to challenge submissions, comments, and other user-generated content.
</Info>

### Reaction Display

Reaction counts show:
- Total count per reaction type
- Your active reactions highlighted
- Click to toggle reaction on/off

```typescript
reactions: {
  reaction_counts: [
    { reaction: "like", count: 24 },
    { reaction: "fire", count: 8 },
    { reaction: "rocket", count: 5 }
  ],
  user_reactions: ["like"]  // User has liked this
}
```

## Submission Gallery

View and showcase completed challenge submissions.

### Challenge Submissions

```typescript
type ChallengeUser = {
  id: number;
  user: User;
  challenge: ChallengeSummary;
  submission_url: string;
  submission_image_url: string;
  reactions: Reactions;
  comments: Comment[];
  completed: boolean;
  submitted_at: string | null;
  listed: boolean;
}
```

### Submission Display

Each submission in the gallery shows:

- **Preview Image**: Screenshot or live preview
- **User Info**: Avatar, name, and badges
- **Submission URL**: Link to deployed project
- **Reactions**: Emoji reaction counts
- **Comments**: Discussion thread
- **Timestamp**: When the submission was created

### Viewing Submissions

```typescript
await getChallengeUsers(request, slug);
// Returns: Array of ChallengeUser submissions
```

<Info>
Submissions can be filtered by completion status, reactions, or date.
</Info>

### Submission Visibility

```typescript
listed: boolean;  // Controls gallery visibility
```

<Note>
Users can choose whether their submission appears in the public gallery.
</Note>
