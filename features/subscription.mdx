## Subscription Plans

Codante.io offers flexible subscription options through the Plan system.

### Plan Structure

Plans are defined with the following properties (from `app/lib/models/plan.server.ts`):

```typescript
type Plan = {
  id: number;
  name: string;                 // e.g., "Pro Monthly", "Pro Yearly"
  price_in_cents: number;       // Price in centavos (R$)
  slug: string;                 // URL-friendly identifier
  details: string;              // HTML description of features
};
```

### Coupon Support

Promotional coupons can be applied during subscription:

```typescript
type Coupon = {
  code: string;                 // Coupon code
  name: string;                 // Display name
  discount_amount: number;      // Discount value
  type: string;                 // "percentage" or "fixed"
  error?: boolean;              // Validation error flag
  message?: string;             // Status message
};
```

## Subscription Management

### Active Subscription

User subscription status is tracked through the Subscription model (from `app/lib/models/subscription.server.ts`):

```typescript
type Subscription = {
  id: string;
  user_id: number;
  status: "active" | "canceled" | "past_due" | "trialing" | "incomplete";
  plan_name: string | null;
  current_period_start: string;
  current_period_end: string;
  cancel_at_period_end: boolean;
  pagarme_order_id: string;
  pagarme_payment_method: string;
  // ... other fields
};
```

### Subscription Status

<Note>
Users with `status: "active"` can access all premium content. Other statuses may have restricted access.
</Note>

### Payment Methods

Codante.io supports multiple payment methods through Pagar.me integration:
- **Credit card** - Instant activation
- **Boleto bancário** - Activation after payment confirmation
- **PIX** - Fast payment with quick activation

## Subscription Workflow

<Steps>
  <Step title="Browse Plans">

## Technical Implementation

### Checking Pro Status

To check if a user has Pro access in your components:

```tsx
import type { User } from "~/lib/models/user.server";
import { useOutletContext } from "react-router";

export default function SomeComponent() {
  const { user } = useOutletContext<{ user: User }>();
  const isPro = user?.subscription?.status === "active";

  return (
    <div>
      {isPro ? (
        <div>Premium content here</div>
      ) : (
        <div>
          <p>This is premium content.</p>
          <a href="/subscription">Upgrade to Pro</a>
        </div>
      )}
    </div>
  );
}
```

### Protected Premium Routes

Server-side protection for premium content:

```typescript
import { requireAuth } from "~/lib/services/auth.server";
import { user } from "~/lib/services/auth.server";
import { redirect } from "react-router";

export async function loader({ request }: LoaderFunctionArgs) {
  await requireAuth(request);
  const currentUser = await user(request);

  // Check if content is premium
  const isPremiumContent = true; // from your content check
  const hasActiveSubscription = 
    currentUser.subscription?.status === "active";

  if (isPremiumContent && !hasActiveSubscription) {
    return redirect("/subscription?required=true");
  }

  // User has access, continue loading content
  return json({ content });
}
```

### Display Pro Badge

Show Pro badge on user profiles and comments:

```tsx
function UserBadge({ user }: { user: User }) {
  const isPro = user.subscription?.status === "active";

  return (
    <div className="flex items-center gap-2">
      <span>{user.name}</span>
      {isPro && (
        <span className="px-2 py-1 text-xs font-semibold bg-indigo-600 text-white rounded">
          PRO
        </span>
      )}
    </div>
  );
}
```

## Best Practices

<Tip>
Always check subscription status server-side for security. Client-side checks are for UX only and can be bypassed.
</Tip>

<Warning>
Cache subscription status carefully. Use short TTLs to ensure users get immediate access after subscribing.
</Warning>

<Info>
Display upgrade prompts contextually when users encounter premium content. Avoid aggressive or intrusive upgrade messaging.
</Info>
