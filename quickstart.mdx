# Quick Start Guide

This guide will help you set up and run the Codante.io platform locally for development.

## Prerequisites

Before you begin, ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Node.js" icon="node-js">
    **Version 20 or higher** is required. Check your version with `node --version`
  </Card>
  
  <Card title="npm" icon="npm">
    Comes bundled with Node.js. Alternatively, you can use yarn.
  </Card>
</CardGroup>

<Warning>
Node.js version 20 or higher is required. The application will not run on older versions.
</Warning>

## Installation

Follow these steps to get your development environment running:

<Steps>
  <Step title="Clone the repository">
    Clone the repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd <project-directory>
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies using npm:
    
    ```bash
    npm install
    ```
    
    This will install all dependencies listed in `package.json`, including:
    - React Router v7 (`react-router`, `@react-router/node`, `@react-router/serve`)
    - React 19.2+
    - TailwindCSS v4
    - Radix UI components
    - Framer Motion for animations
    - And many more development and runtime dependencies
  </Step>

  <Step title="Configure environment variables">
    Create environment configuration files for your local setup. The application uses Zod schema validation for environment variables.
    
    <Info>
    Environment variables are validated using Zod in `app/lib/models/environment.ts`. Required variables include:
    </Info>
    
    **Required Environment Variables:**
    
    ```bash
    # Core Configuration
    SESSION_SECRET=your-session-secret
    API_HOST=http://localhost:8000
    BASE_URL=http://localhost:3000
    NODE_ENV=development
    
    # GitHub OAuth
    GITHUB_ID=your-github-app-id
    GITHUB_SECRET=your-github-app-secret
    GITHUB_CALLBACK_URL=http://localhost:3000/auth/github/callback
    
    # Discord OAuth
    DISCORD_APP_CLIENT_ID=your-discord-client-id
    DISCORD_APP_SECRET=your-discord-secret
    
    # reCAPTCHA
    RECAPTCHA_SITE_KEY=your-recaptcha-site-key
    RECAPTCHA_SECRET_KEY=your-recaptcha-secret-key
    ```
    
    <Note>
    Check `app/lib/models/environment.ts` for the complete list of environment variables and their validation schema.
    </Note>
  </Step>

  <Step title="Start the development server">
    Launch the development server:
    
    ```bash
    npm run dev
    ```
    
    This command uses `react-router dev` which starts:
    - Development server on port **3000** (default)
    - Hot module replacement (HMR) for instant updates
    - Automatic rebuilding on file changes
    
    The application will be available at `http://localhost:3000`
  </Step>

  <Step title="Verify the installation">
    Open your browser and navigate to `http://localhost:3000`. You should see the Codante.io landing page.
    
    <Info>
    In development mode, you'll see a breakpoint indicator in the bottom-left corner showing the current Tailwind breakpoint (xs, sm, md, lg, xl, 2xl).
    </Info>
  </Step>
</Steps>

## Understanding the Root Component

The `app/root.tsx` file is the entry point of the application:

```tsx
export async function loader({ request }: { request: Request }) {
  const userData = (await user({ request })) as User | null;

  // Prefetch optimization
  const headers = new Headers();
  const purpose =
    request.headers.get("Purpose") ||
    request.headers.get("X-Purpose") ||
    request.headers.get("Sec-Purpose") ||
    request.headers.get("Sec-Fetch-Purpose") ||
    request.headers.get("X-Moz");

  if (purpose === "prefetch") {
    headers.set("Cache-Control", "private, max-age=20");
  }

  return data(
    {
      user: userData,
      ENV: {
        BASE_URL: environment().BASE_URL,
        NODE_ENV: environment().NODE_ENV,
      },
    },
    { headers },
  );
}
```

This demonstrates:
- Server-side authentication via `user()` function
- Prefetch optimization with cache headers
- Environment variable exposure to client

## Authentication Flow

The platform supports OAuth authentication with GitHub and Discord:

```tsx
// User data passed via outlet context
const { user } = useOutletContext<{ user: User }>()

// Protected routes use requireAuth() in loaders
// Guest-only routes use requireGuest()
// Session management with cookie-based storage
```

<Info>
Authentication is handled server-side using `remix-auth` with GitHub and Discord strategies. See `app/lib/services/auth.server.ts` for implementation details.
</Info>

## Styling with TailwindCSS

The application uses TailwindCSS v4 with a custom design system:

```tsx
import { cn } from "~/lib/utils/cn";

// Use cn() utility (clsx + tailwind-merge) for conditional classes
<div className={cn(
  "text-gray-800 bg-white",
  "dark:bg-gray-dark dark:text-gray-50"
)} />
```

### Styling Conventions

- **cn() utility** - Combines clsx and tailwind-merge for conditional classes
- **Dark mode** - Via ColorModeProvider context and CSS custom properties
- **Component variants** - Using class-variance-authority
- **Background patterns** - Custom patterns like `bg-grainy`
- **Responsive design** - Mobile-first with Tailwind breakpoints

## Troubleshooting

### Port Already in Use

If port 3000 is already in use, the development server will fail to start. Either stop the process using that port or configure a different port.

### Environment Variables Missing

If you see Zod validation errors, ensure all required environment variables are set correctly. Check `app/lib/models/environment.ts` for the validation schema.

### Type Errors

Run `npm run typecheck` to identify and fix TypeScript errors. The command also generates route types.

<Warning>
Always run type checking before committing code. The project uses Git hooks to enforce code quality standards.
</Warning>
